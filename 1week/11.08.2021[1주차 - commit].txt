Git 핵심 쏙쏙

[1주차]

✔ commit

- git은 버전관리 도구이다. 언제 어디서 하던 어떤 파일을 업로드 하였는지 부가설명이라고 생각하면된다.
- 프로젝트별로 어떤 작업을 했는지도 알려주는 작업이다.
- 버전별로 만들필요 없이 중간중간 git을 사용해 현재 프로젝트의 상태만 저장하면 된다.
- git은 누가, 언제, 현재 프로젝트 상태를 저장한 것을 commit이라고 표현한다.
- 현재 프로젝트의 상태를 저장하고 snapshot을 찍은 것 처럼 프로젝트의 전체 상태를 말하는 것이다.

✔ 정리

- 버전관리를 한다는 것은 프로젝트 상태가 변경되는 정보를 알고 있다는 것이다. Git은 commit을 사용해서
 버전이 달라지는 것을 관리하는 역할이다.
- 컴퓨터에 있는 프로젝트를 Git이 관리하는 프로젝트로 만들 수 있다. Git으로 관리하고 설정하는 것을
 git initialize(git 초기화)한다고 표현한다.
- 또한 현재 프로젝트의 상태를 사진찍어서 저장하는 것을 commit 한다라고도 한다.
- commit은 누가(author), 언제 commit 했는지의 정보와 프로젝트를 변경했고, 작업내역이 어떤 것인지
 알아 볼 수 있게 저는 메시지를 'commit 메시지'라고 한다.
- commit에 반영할지 안할지는 파일단위로 선택할 수 있다. commit에 반영할 파일을 선택하는 것을 add
 (혹은 staging, 스테이징)이라고 한다.
- commit한 기록은 history로 볼 수 있다.
- 현재까지 작업을 'git initialize -add(staging) -commit'이라고 한다.
- git initialize는 처음에 단 한번만 해주면 된다. 작업내역을 저장하기 위해서는 add -commit만 하면된다.
